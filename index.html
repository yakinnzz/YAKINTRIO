💡 PENGERTIAN POLIMORFISME & ABSTRAKSI

Polimorfisme (banyak bentuk) - objek dapat diperlakukan sebagai bentuk yang berbeda.

Abstraksi - menyembunyikan detail implementasi, hanya menampilkan fungsi penting.

1. PENGERTIAN PEWARISAN extends

Penjelasan: Inheritance memungkinkan class child mewarisi properti dan method dari class parent.

// Nama  : USMAN YAKIN TRIO SINADA
// NIM   : 251410006
// Kelas : SI2A

class Hewan {
  void bersuara() => print('...');
}

class Anjing extends Hewan {
  @override
  void bersuara() => print('Guk Guk');
}

void main() {
  Hewan hewan = Anjing();
  hewan.bersuara(); // Output: Guk Guk
}

3. POLIMORFISME DENGAN INHERITANCE

Penjelasan: Referensi tipe parent dapat menampung objek child, memungkinkan satu tipe data untuk berbagai objek.

class Bentuk {
  void gambar() => print('Menggambar bentuk');
}

class Persegi extends Bentuk {
  @override void gambar() => print('Menggambar Persegi');
}

class PersegiPanjang extends Bentuk {
  @override void gambar() => print('Menggambar Persegi Panjang');
}

void main() {
  Bentuk b1 = Persegi();
  Bentuk b2 = PersegiPanjang();
  b1.gambar(); // Menggambar Persegi
  b2.gambar(); // Menggambar Persegi Panjang
}

4. MANFAAT POLIMORFISME

Penjelasan: Fleksibel, mudah dikembangkan, kode dapat digunakan ulang, dan perawatan lebih mudah.

  • ✅ Fleksibilitas untuk berbagai jenis objek
  • ✅ Penambahan tipe baru tanpa mengubah banyak kode
  • ✅ Penggunaan ulang method dari class induk
  • ✅ Kemudahan perawatan program

5. IS DAN AS OPERATOR

Penjelasan: is untuk memeriksa tipe objek, as untuk mengubah tipe objek.

class Transportasi {}
class Bus extends Transportasi {
  void klakson() => print('Bip bip!');
}

void main() {
  Bus k = Bus();
  k.klakson(); // Output: Bip bip!
}

6. PRAKTIK POLIMORFISME DENGAN LIST

Penjelasan: List dengan tipe parent dapat menyimpan berbagai objek subclass. Perulangan akan menjalankan method sesuai implementasi masing-masing.

class Hewan { void bersuara() => print('Hewan bersuara'); }
class Kucing extends Hewan { @override void bersuara() => print('Meong'); }
class Anjing extends Hewan { @override void bersuara() => print('Guk guk'); }
class Burung extends Hewan { @override void bersuara() => print('Cuit cuit'); }

void main() {
  List daftarHewan = [Kucing(), Anjing(), Burung()];
  for (var h in daftarHewan) { h.bersuara(); }
}

  • 🔹 Menyederhanakan sistem
  • 🔹 Mengurangi kompleksitas

8. ABSTRACT CLASS

Penjelasan: Class blueprint yang tidak bisa diinstansiasi langsung, hanya bisa digunakan melalui class turunan.

abstract class Kendaraan {
  String? merk;
  void start();
  void stop() => print('Kendaraan berhenti');
}

class Mobil extends Kendaraan {
  @override void start() => print('Mobil $merk menyala');
}

void main() {
  Mobil m = Mobil();
  m.merk = "BMW";
  m.start(); // Mobil BMW menyala
  m.stop();  // Kendaraan berhenti
}

9. ABSTRACT METHOD

Penjelasan: Method tanpa implementasi yang wajib di-override oleh class turunan.

abstract class AlatMusik {
  void mainkan(); // abstract method
  void info() => print('Ini adalah alat musik');
}

class Gitar extends AlatMusik {
  @override void mainkan() => print('Jreng jreng...');
}

void main() {
  Gitar g = Gitar();
  g.info();   // Ini adalah alat musik
  g.mainkan(); // Jreng jreng...
}

10. INTERFACE DI DART

Penjelasan: Menggunakan implements, class wajib mengimplementasikan semua method dari interface.

interface class Flyable {
  void terbang() => print('Terbang...');
}

class KupuKupu implements Flyable {
  @override void terbang() => print('Kupu-kupu terbang indah');
}

class Drone implements Flyable {
  @override void terbang() => print('Drone terbang cepat');
}

void main() {
  KupuKupu k = KupuKupu();
  Drone d = Drone();
  k.terbang(); // Kupu-kupu terbang indah
  d.terbang(); // Drone terbang cepat
}

11. PERBEDAAN EXTENDS VS IMPLEMENTS

Penjelasan: extends mewarisi implementasi, implements hanya mewarisi kontrak (wajib override semua).

// extends: bisa menggunakan atau override method dari parent
// implements: wajib override SEMUA method, tidak mewarisi implementasi

12. STATIC MEMBERS DAN METHOD

Penjelasan: Static member bisa diakses langsung tanpa membuat objek, cukup dengan nama class.

class MathUtils {
  static double pi = 3.14;
  static int add(int x, int y) => x + y;
}

void main() {
  print('Nilai pi : ${MathUtils.pi}');      // 3.14
  print('Result : ${MathUtils.add(10, 20)}'); // 30
}

13. PRAKTIK - SISTEM PEMBAYARAN

Penjelasan: Implementasi interface untuk berbagai metode pembayaran (Kartu Kredit, E-Wallet, Tunai).

abstract class MetodePembayaran { void bayar(double jumlah); }

class KartuKredit implements MetodePembayaran {
  @override void bayar(double jumlah) => print('Bayar $jumlah dengan Kartu Kredit');
}

class EWallet implements MetodePembayaran {
  @override void bayar(double jumlah) => print('Bayar $jumlah dengan E-Wallet');
}

class Tunai implements MetodePembayaran {
  @override void bayar(double jumlah) => print('Bayar $jumlah dengan Tunai');
}

void main() {
  MetodePembayaran m1 = KartuKredit();
  MetodePembayaran m2 = EWallet();
  MetodePembayaran m3 = Tunai();
  m1.bayar(100000);
  m2.bayar(50000);
  m3.bayar(20000);
}

14. STUDI KASUS - BENTUK GEOMETRI

Penjelasan: Sistem menghitung luas dan keliling berbagai bangun (Persegi, Lingkaran) dengan abstract class.

abstract class Bentuk {
  double hitungLuas();
  double hitungKeliling();
}

class Persegi implements Bentuk {
  double sisi;
  Persegi(this.sisi);
  @override double hitungLuas() => sisi * sisi;
  @override double hitungKeliling() => 4 * sisi;
}

class Lingkaran implements Bentuk {
  double radius;
  Lingkaran(this.radius);
  @override double hitungLuas() => 3.14 * radius * radius;
  @override double hitungKeliling() => 2 * 3.14 * radius;
}

void main() {
  Bentuk b1 = Persegi(4);
  Bentuk b2 = Lingkaran(7);
  print('Persegi: luas=${b1.hitungLuas()}, keliling=${b1.hitungKeliling()}');
  print('Lingkaran: luas=${b2.hitungLuas()}, keliling=${b2.hitungKeliling()}');
}
